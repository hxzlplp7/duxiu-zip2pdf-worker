## 📊 目录说明

| 文件 | 说明 | 大小 |

| `index.js` | Worker主文件，包含所有业务逻辑 | ~38KB |
| `passwords.js` | 391个常用密码字典 | ~8KB |

**特性**：
- ✨ 完整的ZIP转PDF转换逻辑

- 📖 封面封底智能识别
- 📤 分块文件上传
- ⏳ 24小时自动删除
- ⚠️ 免责声明展示

## 🎯 为什么这样组织？

### ✅ 优点

1. **清晰分离**
   - 代码在 `src/`
   - 文档在 `docs/`
   - 配置在根目录

2. **易于维护**
   - 文档集中管理
   - 代码结构简单
   - 配置一目了然

3. **标准化**
   - 符合开源项目规范
   - GitHub自动识别docs/
   - npm标准结构

4. **可扩展**
   - 添加新文档到docs/
   - 添加新代码到src/
   - 不污染根目录

## 💡 最佳实践

### 添加新文档
```bash
# 在docs/目录创建文档
touch docs/新功能说明.md

# 在README.md中添加链接
echo "- [新功能说明](./docs/新功能说明.md)" >> README.md
```

### 添加新代码
```bash
# 在src/目录创建新文件
touch src/utils.js

# 在index.js中导入
echo "import { xxx } from './utils.js';" >> src/index.js
```

### 更新文档
```bash
# 直接编辑docs/目录中的文件
code docs/使用说明.md

# 提交更改
git add docs/
git commit -m "Update: 更新使用说明"
git push
```
